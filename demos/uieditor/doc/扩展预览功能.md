# SouiEditor 扩展预览功能文档

## 概述

SouiEditor 提供了灵活的扩展预览机制，支持两种方式来处理项目中自定义的控件和皮肤：

1. **插件模式** - 通过加载外部扩展控件库（`.exctrl` 文件）来注册自定义控件
2. **配置映射模式** - 通过配置文件将未注册的控件/皮肤映射到其基类进行预览

---

## 一、插件模式注册自定义控件

### 1.1 原理说明

SouiEditor 在启动时会自动扫描应用程序目录下所有 `.exctrl` 后缀的动态库文件，并调用其中导出的 `RegisterExCtrls` 函数来注册扩展控件。

相关代码位于 [SouiEditor.cpp](../SouiEditor.cpp#L136-L159)：

```cpp
#ifdef SOUI_ENABLE_CORE_LIB
// 注册扩展控件
SCtrlsRegister::RegisterCtrls(&theApp);
#else
// 加载扩展控件
WIN32_FIND_DATA fd;
HANDLE hFind = FindFirstFile((appDir + _T("/*.exctrl")).c_str(), &fd);
if(hFind != INVALID_HANDLE_VALUE)
{
    do
    {
        HMODULE hExCtrl = LoadLibrary(fd.cFileName);
        if(hExCtrl)
        {
            FUN_REG_EXCTRLS pfnReg = (FUN_REG_EXCTRLS)GetProcAddress(hExCtrl, "RegisterExCtrls");
            if(pfnReg)
            {
                pfnReg(&theApp);
            }
        }
    }while(FindNextFile(hFind, &fd));
    FindClose(hFind);
}
#endif//SOUI_ENABLE_CORE_LIB
```

### 1.2 创建扩展控件库

#### 步骤 1：定义插件接口

插件接口定义位于 [include/plugin-i.h](../include/plugin-i.h)：

```cpp
#ifndef __PLUGIN_I_H__
#define __PLUGIN_I_H__
#include <SApp.h>

SNSBEGIN
typedef BOOL (* FUN_REG_EXCTRLS)(SApplication *pApp);

EXTERN_C BOOL  SOUI_COM_API RegisterExCtrls(SApplication *pApp);

SNSEND    
#endif//__PLUGIN_I_H__
```

#### 步骤 2：实现扩展控件库

参考 [exctrls/exctrls.cpp](../exctrls/exctrls.cpp) 的实现：

```cpp
#include <souistd.h>
#include "../include/plugin-i.h"
#include <SCtrlsRegister.h>
SNSBEGIN

SOUI_COM_C BOOL SOUI_COM_API RegisterExCtrls(SApplication *pApp)
{
    SCtrlsRegister::RegisterCtrls(pApp);
    return TRUE;
}

SNSEND
```

#### 步骤 3：编译并部署

1. 将编译生成的 DLL 文件重命名为 `.exctrl` 后缀
2. 将 `.exctrl` 文件放置到 SouiEditor 应用程序目录下
3. 启动 SouiEditor 时会自动加载并注册其中的控件

### 1.3 注意事项

- 只有在SOUI核心控件库编译为动态库时，才能使用插件模式, 静态库不支持插件模式
- 动态库必须导出 `RegisterExCtrls` 函数，否则会被忽略
- 使用 `SOUI_COM_API` 宏确保函数正确导出
- 参考 [exctrls/exctrls.cpp](../exctrls/exctrls.cpp) 中的实现

## 二、配置映射模式（未注册控件/皮肤预览）

对于没有在扩展库中注册的自定义控件或皮肤，SouiEditor 支持通过配置文件将其映射到已知的父类，从而在预览时自动创建为父类类型。

### 2.1 原理说明

SouiEditor 通过重载 [SApplication](../SouiEditor.cpp#L53-L89) 的 `CreateWindowByName` 和 `CreateSkinByName` 方法实现：

```cpp
class CUiEditorApp : public SApplication
{
public:
    IWindow *CreateWindowByName(LPCWSTR pszWndClass) const override{
        IWindow *pRet = SApplication::CreateWindowByName(pszWndClass);
        if(pRet)
            return pRet;
        // 从配置中查找自定义控件的父类
        SStringW strParent = CSysDataMgr::getSingleton().GetUserWidgetParent(pszWndClass);
        if(!strParent.IsEmpty())
        {
            return SApplication::CreateWindowByName(strParent);
        }
        return NULL;
    }

    ISkinObj *CreateSkinByName(LPCWSTR pszSkinClass) const override{
        ISkinObj *pRet = SApplication::CreateSkinByName(pszSkinClass);
        if(pRet)
            return pRet;
        // 从配置中查找自定义皮肤的父类
        SStringW strParent = CSysDataMgr::getSingleton().GetUserSkinParent(pszSkinClass);
        if(!strParent.IsEmpty())
        {
            return SApplication::CreateSkinByName(strParent);
        }
        return NULL;
    }
};
```

### 2.2 配置自定义控件映射

编辑 [Config/Ctrl.xml](../Config/Ctrl.xml#L4-L7) 文件，在 `<user_widgets>` 节点下添加映射：

```xml
<user_widgets>
    <!-- 用户没有注册的自定义控件，自动识别为它的父类型 -->
    <!-- checkgroup为用户自定义的类型, parent为父类型, 编辑时自动创建为父类型 -->
    <checkgroup parent="window"/>
    
    <!-- 示例：将自定义控件 myButton 映射到 button -->
    <myButton parent="button"/>
    
    <!-- 示例：将自定义控件 myPanel 映射到 window -->
    <myPanel parent="window"/>
</user_widgets>
```

### 2.3 配置自定义皮肤映射

编辑 [Config/Skin.xml](../Config/Skin.xml#L18-L21) 文件，在 `<user_skins>` 节点下添加映射：

```xml
<user_skins>
    <!-- 项目中使用到的未注册的皮肤对象，在这里配置，用它的父对象来代替 -->
    <test_skin parent="imglist"/>
    
    <!-- 示例：将自定义皮肤 myButtonSkin 映射到 imglist -->
    <myButtonSkin parent="imglist"/>
    
    <!-- 示例：将自定义皮肤 myFrameSkin 映射到 imgframe -->
    <myFrameSkin parent="imgframe"/>
</user_skins>
```

### 2.4 配置说明

| 属性 | 说明 |
|------|------|
| 节点名 | 自定义控件/皮肤的名称（在 XML 布局中使用） |
| `parent` | 父类名称，必须是 SOUI 内置或已注册的控件/皮肤类型 |

### 2.5 常用父类参考

**控件父类：**
- `window` - 基础窗口控件
- `button` - 按钮控件
- `check` - 复选框
- `radio` - 单选框
- `edit` - 编辑框
- `text` - 文本控件
- `imgbtn` - 图片按钮
- `scrollview` - 滚动视图

**皮肤父类：**
- `imglist` - 图片列表皮肤
- `imgframe` - 图片边框皮肤
- `imgcenter` - 图片居中皮肤
- `gradation` - 渐变皮肤
- `colorrect` - 颜色矩形皮肤

## 三、使用建议

### 场景 1：项目中有自定义控件库

如果项目中有自定义的 SOUI 控件库，建议：

1. 创建一个扩展控件项目，实现 `RegisterExCtrls` 函数
2. 在函数中注册项目中的所有自定义控件
3. 编译为 `.exctrl` 文件并放入 SouiEditor 目录
4. 这样可以获得完整的控件属性和预览支持

### 场景 2：少量未注册控件/皮肤

如果只有少量未注册的自定义控件或皮肤，建议：

1. 在 `Config/Ctrl.xml` 和 `Config/Skin.xml` 中配置映射
2. 选择合适的父类，确保预览效果最接近实际效果
3. 这种方式简单快捷，无需编写扩展库

### 场景 3：混合使用

也可以同时使用两种方式：
- 将主要自定义控件编译为扩展库
- 在配置文件中映射一些临时的或简单的自定义类型

## 四、预览功能实现

SouiEditor 的预览功能由 [CPreviewHost](../designer/PreviewHost.h) 类实现，支持：

- 命令行启动预览模式：`uieditor -preview <项目路径> -layout <布局ID>`
- 菜单启动预览模式：`菜单->预览`
